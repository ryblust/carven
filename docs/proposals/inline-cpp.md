# 内联 C++ 代码

## 目标

在 Carven 源文件中直接嵌入原始 C++ 代码，作为转译过程中的"逃生出口"——当 Carven 语法无法表达某些 C++ 特性时使用。

## 候选语法

### 选项 A：C 风格预处理器指令

```carven
#cpp {
    // 原始 C++ 代码，直接原样输出到生成的 C++ 文件中
    auto lambda = [](int x) { return x * 2; };
}
```

### 选项 B：Rust 式属性

```carven
#[cpp] {
    auto lambda = [](int x) { return x * 2; };
}
```

### 选项 C：C++ 式属性

```carven
[[cpp]] {
    auto lambda = [](int x) { return x * 2; };
}
```

## 对比

| 维度 | A (`#cpp`) | B (`#[cpp]`) | C (`[[cpp]]`) |
|------|------------|--------------|---------------|
| 简洁性 | 高 | 中 | 低 |
| 与 C/C++ 预处理器风格一致 | 是 | 否 | 否 |
| 可以嵌入表达式（非块） | `#cpp(x + y)` | `#[cpp](x + y)` | 不自然 |
| 与 inline test 语法的一致性 | 高（如果选 `#test`） | 高（如果选 `#[test]`） | 高（如果选 `[[test]]`） |

## 使用场景

- 调用 C++ 模板/概念/requires 子句等 Carven 不直接支持的语法
- 嵌入 lambda 表达式（Carven 还不支持 lambda）
- 使用 C++ 特有的库 API
- 性能关键路径上的手写 C++ 优化

## 注意事项

- 内联 C++ 代码**不经过** Carven 的词法/语法分析——直接嵌入输出
- 内联 C++ 中的大括号 `{}` 必须正确配对（否则破坏转译输出的结构）
- 滥用内联 C++ 会削弱 Carven 的价值——它应该是最后手段
