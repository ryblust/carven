# Match 表达式标识符的二义性

match 模式中，标识符总是被当作**类型模式**处理，不能匹配变量值。这是上下文无关解析器的硬性限制——parser 区分不了标识符是类型名还是变量名。

```carven
match x {
    i32  => { }   // 类型匹配: std::is_same_v<_, std::int32_t>
    zero => { }   // 也是类型匹配，不是值匹配
    _    => { }   // 通配符
}
```

用户直觉上可能认为 `zero` 匹配变量 `zero` 的值，但实际不会。

## 修复

在 `docs/grammar.md` 的 Match 表达式章节末尾加一段说明，解释：

> 标识符模式始终被视为类型匹配，不能用于匹配变量值。值匹配只支持字面量（`42`、`true`、`"hello"`）。需要值匹配变量时，请使用 `if` 表达式代替。
